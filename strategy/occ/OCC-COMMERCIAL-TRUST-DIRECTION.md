# OCC Commercial Trust Direction

**Status:** Current internal PRS direction.

The OCC repository remains authoritative for product and technical implementation.

## Sellable Alpha

Sellable alpha means a user can pay because OCC saves meaningful time and generates real outputs worth paying for.

Internal quality bar: a user should be able to request a bounded build, such as a creatively distinct simple game based on a reference, and OCC should materially progress or complete it autonomously while the user is away.

This is internal direction, not a finalized v1 specification, public capability claim, or statement that the capability is already complete.

## Managed Usage Direction

Preferred direction:

- One OCC account and one setup.
- Managed multi-provider access.
- Optional customer API keys.
- Optional local AI.
- Clear customer-controlled budget.

This is not yet locked into public packaging or pricing.

Customer-owned API/local capacity should reduce OCC's fee/take for that usage and may receive user-selected routing preference.

Routing must optimize for customer budget, quality, outcome, and stated preferences rather than undisclosed OCC-margin optimization.

## Launch-Critical Budget Controls

Launch needs:

- Visible spend.
- Forecasts and explanations.
- User-set limits.
- Project budgets.
- Task budgets.
- Workspace budgets.
- Approval gates for costly work.
- Alerts.
- Hard stops.
- Kill-switch behavior.

No unlimited managed frontier-model promise.

## Security Direction

Security should be developed as costed phases:

1. Founder/alpha baseline.
2. Launch hardening.
3. High-security local/infrastructure operations.

Current operational-security gaps include password management and broader operational security maturity.

## Evidence Needed

Before pricing, forecasts, external security language, or investor claims can be finalized, PRS needs:

- Verified model/provider pricing and usage costs.
- Infrastructure, support, and tool costs.
- Actual founder and business costs.
- Legal, tax, accounting, and security estimates.
- Customer-acquisition assumptions and evidence.
- Competitive pricing evidence.
- Final OCC packaging and launch offer.
- Costed budget-control and security-phase plans.
